IMPORTANT — how to pick `cwe_id`:

NVD assigns the CWE that describes the **security consequence** (the effect an
attacker can achieve), not the CWE that describes the code-level mechanism
(the bug pattern in the diff). Pick the **effect-level** CWE.

Examples of the distinction:
  - Diff adds a cycle break in a certificate-chain walk.
    Mechanism: CWE-835 (loop with no exit).  ← do not pick this.
    Effect:    CWE-347 (improper signature verification).  ← pick this.
  - Diff strips control characters from user-supplied metadata.
    Mechanism: CWE-74 (injection).            ← do not pick this.
    Effect:    CWE-400 (resource exhaustion).  ← pick this if that's the
               advisory's impact.

Rules:
  1. Read the diff for mechanism, then ask "what can an attacker do with
     this — what is the *resulting* weakness?". That is your `cwe_id`.
  2. If mechanism and effect are the same (e.g. a plain heap overflow that
     just corrupts memory), use the shared CWE.
  3. Prefer the most specific effect CWE that would appear on an NVD record.

Respond with a single JSON object and nothing else. Schema:
