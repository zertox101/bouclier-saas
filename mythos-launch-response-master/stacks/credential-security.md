# Credential Security: Passwords, MFA, Banking, and Account Protection

**The #1 thing you can do to protect your business. Before anything else, do this.**

If an attacker gets your password, they get your account. If they get your email password, they get EVERY account (via password resets). If they get your banking credentials, they get your money. Mythos-class AI makes credential attacks cheaper and faster. This guide makes you harder to crack.

---

## 1. Password Security

### The Rules (Non-Negotiable)

| Rule | Why |
|------|-----|
| **Minimum 16 characters** | Shorter passwords can be brute-forced. 16+ characters are computationally infeasible to crack with current hardware. |
| **Every account gets a unique password** | When one service gets breached, attackers try that password on every other service. Reuse = all accounts compromised at once. |
| **Never use personal information** | Your name, birthday, pet's name, address — attackers check these first. AI makes guessing personal passwords trivial. |
| **Never share passwords** | Not with coworkers, not with IT, not with "support." Legitimate services never ask for your password. |
| **Never type passwords into links from emails** | Always navigate to the site directly. AI-generated phishing is now indistinguishable from legitimate email. |

### What Makes a Strong Password

```
BAD:
  Password123!        (dictionary word + predictable pattern)
  Summer2026!         (seasonal + year + symbol = predictable)
  CompanyName1        (attacker's first guess)
  J0hn$mith99         (personal info with substitutions)

GOOD (passphrase method):
  correct-horse-battery-staple-mountain    (5 random words, easy to remember)
  purple-freight-telescope-canyon-eleven   (5 random words)

BEST (generated by password manager):
  x7#mK9$pL2@nF5vR   (random, stored in password manager)
```

**The answer is a password manager.** You don't need to remember strong passwords — your password manager remembers them for you. You only remember ONE strong master password.

### Password Managers

| Manager | Cost | Best For |
|---------|------|----------|
| **Bitwarden** | Free personal, $4/user/mo teams | Best free option. Open source. Can self-host. |
| **1Password** | $8/user/mo business | Best team management. Excellent onboarding. |
| **KeePassXC** | Free | Local-only storage. No cloud. Maximum control. |

**Deployment for a business:**

1. [ ] Choose a password manager (Bitwarden recommended for SMBs)
2. [ ] Create the organization/team account
3. [ ] Invite all employees
4. [ ] **Require** all employees to install and use it (not optional)
5. [ ] Create shared vaults for team passwords (Wi-Fi, shared accounts)
6. [ ] Set the master password policy: 16+ characters, passphrase recommended
7. [ ] Enable MFA on the password manager itself (this protects the keys to the kingdom)
8. [ ] Store the emergency access / recovery kit in a physical safe

### Password Rotation Policy

| Account Type | Rotation Frequency | Trigger for Immediate Rotation |
|-------------|-------------------|-------------------------------|
| **Password manager master password** | Every 12 months | Any suspicion of compromise |
| **Email accounts** | Every 12 months | Any breach notification, any suspicious activity |
| **Banking / financial** | Every 6 months | Any suspicious transaction, any breach |
| **Admin accounts** (IT systems, cloud) | Every 90 days | Staff departure, any incident |
| **API keys and secrets** | Every 90 days | Staff departure, any incident, any key exposure |
| **Wi-Fi password** | Every 6 months | Staff departure, shared with guests |
| **Standard user accounts** | When compromised | Breach notification from HIBP |

**On any employee departure, immediately rotate:**
- [ ] All passwords the employee knew or could access
- [ ] All shared passwords (Wi-Fi, shared accounts)
- [ ] All API keys the employee had access to
- [ ] Revoke their access to the password manager

### Check If Your Passwords Are Already Compromised

1. Go to **haveibeenpwned.com**
2. Enter every business email address
3. If any appear in breaches — change those passwords immediately
4. Enable HIBP notifications for ongoing monitoring

**Bitwarden and 1Password both integrate with HIBP** — they'll automatically flag any stored passwords that appear in known breaches.

## 2. Multi-Factor Authentication (MFA)

### What MFA Is

MFA requires two different proofs of identity:
- **Something you know** (password)
- **Something you have** (phone, hardware key)
- **Something you are** (fingerprint, face)

Even if an attacker steals your password, they can't log in without the second factor.

### MFA Methods — Ranked by Security

| Method | Security | Convenience | Use When |
|--------|:--------:|:-----------:|----------|
| **Hardware security key** (YubiKey, Titan) | Highest | Medium | Admin accounts, banking, email, password manager |
| **Authenticator app** (Google/Microsoft Authenticator, Authy) | High | High | All accounts that support it |
| **Push notification** (Microsoft Authenticator push, Duo) | High | Highest | Daily-use accounts |
| **SMS text code** | Low | High | **Only as last resort** — SIM swapping attacks can intercept SMS |
| **Email code** | Low | Medium | **Only as last resort** — if email is compromised, MFA is bypassed |
| **Security questions** | NOT MFA | — | **Never rely on these** — answers are guessable or on social media |

### Where to Enable MFA (Priority Order)

**CRITICAL — Do these today:**

1. [ ] **Email** (this is #1 — email resets every other password)
   - Gmail: myaccount.google.com → Security → 2-Step Verification
   - Outlook/M365: mysignins.microsoft.com → Security info → Add method
   - Yahoo: login.yahoo.com → Account Security → Two-step verification

2. [ ] **Password manager** (protects all your other passwords)
   - Bitwarden: Settings → Two-step Login
   - 1Password: My Profile → More Actions → Manage Two-Factor Authentication

3. [ ] **Banking and financial services** (protects your money)
   - Log into each bank/financial account → Security Settings → enable MFA
   - Use authenticator app, NOT SMS, if the bank supports it

4. [ ] **Cloud and hosting accounts** (protects your infrastructure)
   - AWS: IAM → Users → Security Credentials → Assign MFA
   - Google Cloud: cloud.google.com → Security
   - Azure: portal.azure.com → Azure Active Directory → MFA
   - Vercel, Supabase, etc.: Account settings → Security

5. [ ] **Domain registrar** (protects your website and email)
   - If an attacker controls your domain, they control your email, your website, and every service that uses your domain
   - GoDaddy, Namecheap, Cloudflare — enable MFA in account settings

**HIGH — Do this week:**

6. [ ] **Social media accounts** (prevents business impersonation)
7. [ ] **Accounting / ERP software** (QuickBooks, Xero, etc.)
8. [ ] **VPN or remote access**
9. [ ] **Code repositories** (GitHub, GitLab, Bitbucket)
10. [ ] **CRM** (Salesforce, HubSpot, etc.)

**Everything else:** Enable MFA on every account that supports it. No exceptions.

### MFA Enforcement for Teams

If you manage a team, don't make MFA optional:

**Google Workspace:**
- Admin console → Security → 2-Step Verification → Enforcement → On

**Microsoft 365:**
- Azure AD → Security → MFA → Require for all users
- Or: Conditional Access → New policy → Require MFA for all users, all apps

**GitHub Organization:**
- Organization Settings → Authentication security → Require two-factor authentication

### Hardware Security Keys — The Gold Standard

For anyone with access to financial systems, admin accounts, or sensitive data:

| Key | Cost | Works With |
|-----|------|-----------|
| **YubiKey 5 NFC** | ~$50 | USB-A + NFC (phones). Works with everything. |
| **YubiKey 5C NFC** | ~$55 | USB-C + NFC. For newer laptops. |
| **YubiKey 5Ci** | ~$75 | USB-C + Lightning. For iPhone users. |
| **Google Titan** | ~$30-35 | USB-A/C + NFC. Good budget option. |

**Buy TWO keys per person.** One for daily use, one as backup stored in a safe. If you lose your only key, you're locked out.

Registration:
1. Log into the account → Security → Add security key
2. Insert the key → tap it when it blinks
3. Register the backup key too
4. Store backup key in a physical safe

## 3. Banking and Financial Account Security

### Online Banking

- [ ] **MFA enabled** — authenticator app or hardware key, NOT SMS if avoidable
- [ ] **Unique password** — never reuse your banking password anywhere
- [ ] **Transaction alerts** — set up text/email alerts for ALL transactions (even small ones — attackers test with small amounts first)
- [ ] **Daily balance monitoring** — check your accounts every business day
- [ ] **Positive pay / ACH filters** — ask your bank about fraud prevention services:
  - **Positive pay:** Bank only honors checks that match a list you provide
  - **ACH debit block/filter:** Prevents unauthorized ACH withdrawals
  - **Wire transfer callbacks:** Bank calls you to verify wire requests
- [ ] **Dual authorization for wire transfers** — require two people to approve any wire
- [ ] **IP restrictions** — some business banking portals allow restricting login to specific IPs
- [ ] **Session timeout** — ensure your banking session logs out after inactivity (don't leave it open)
- [ ] **Dedicated banking device** — if practical, access banking ONLY from one trusted, hardened computer (not the same machine used for email and web browsing)

### Business Credit Cards

- [ ] **Individual cards per employee** — no shared card numbers
- [ ] **Spending limits per card** — each card should have the minimum necessary limit
- [ ] **Real-time transaction alerts** — on every card
- [ ] **Virtual card numbers** — for online purchases, use virtual/disposable card numbers if your provider offers them
- [ ] **Disable cards** immediately when employees leave

### Accounting Software (QuickBooks, Xero, etc.)

- [ ] **MFA enabled** for all users
- [ ] **Unique passwords** for each user (no shared logins)
- [ ] **User roles reviewed** — does the intern need admin access? Probably not.
- [ ] **Audit trail enabled** — log all changes to financial records
- [ ] **Bank feed connections secured** — review which bank accounts are connected and who has access
- [ ] **Automatic backups** — export data regularly in case the cloud service is compromised
- [ ] **Access removed immediately** when employees or contractors leave
- [ ] **Accountant access** — use the dedicated accountant access feature rather than sharing admin credentials

### Payroll Systems

- [ ] **MFA enabled** — payroll fraud is a major attack target
- [ ] **Dual authorization** for any changes to bank routing numbers (attackers redirect direct deposits)
- [ ] **Employee verification** for any change to direct deposit info — call the employee at a known number to confirm
- [ ] **Separation of duties** — the person who enters payroll shouldn't be the same person who approves it
- [ ] **Regular audit** — review payroll reports for ghost employees or unauthorized changes

### Tax and IRS Credentials

- [ ] **IRS e-Services password** secured with MFA
- [ ] **EFIN/PTIN credentials** — unique, strong passwords, MFA enabled
- [ ] **e-File credentials** never shared via email
- [ ] **CAF number** — monitor for unauthorized use
- [ ] **IRS Identity Protection PIN** — consider enrolling for yourself and advising clients

## 4. Account Recovery Planning

### What Happens If You're Locked Out?

For each critical account, document:

| Account | Recovery Method | Backup Method | Where Recovery Info Is Stored |
|---------|----------------|---------------|------------------------------|
| Password manager | Master password + recovery kit | Emergency contact access | Physical safe |
| Email | Recovery email + phone | Backup codes | Password manager + physical safe |
| Banking | Branch visit with ID | Phone banking PIN | Password manager |
| Domain registrar | Recovery email + MFA | Registrar support + ID verification | Password manager |

### Recovery Codes

Most services that offer MFA also provide one-time recovery/backup codes. These are your "break glass in emergency" access.

- [ ] **Generate recovery codes** for every MFA-protected account
- [ ] **Print them** — store printed copies in a physical safe
- [ ] **Also store in password manager** — as a backup to the physical copy
- [ ] **Never store recovery codes ONLY on your phone** — if you lose the phone, you lose both MFA and recovery

### Emergency Access

- [ ] **Password manager emergency access** configured (Bitwarden and 1Password both support this):
  - Designate a trusted person who can request access after a waiting period
  - The waiting period gives you time to deny the request if it's unauthorized
  - If you're incapacitated, the trusted person gains access after the wait
- [ ] **Documented succession plan** for digital accounts — if you're unavailable, who can access what?
- [ ] **Physical safe** contains: recovery codes, master password hint, list of critical accounts, emergency contacts

## 5. Monitoring and Breach Response

### Ongoing Monitoring

- [ ] **Have I Been Pwned notifications** — register every business email at haveibeenpwned.com for free alerts
- [ ] **Password manager breach alerts** — Bitwarden and 1Password flag passwords found in breaches
- [ ] **Banking transaction alerts** — immediate notification on all activity
- [ ] **Login alerts** — enable "new device" or "new location" alerts on email, cloud services, and admin accounts
- [ ] **Dark web monitoring** — some password managers and EDR tools include this

### If You Suspect a Credential Compromise

**Immediate actions (do all of these, in order):**

1. [ ] **Change the compromised password** — from a DIFFERENT device than the one that may be compromised
2. [ ] **Check for unauthorized changes** — forwarding rules on email, new admin users, changed recovery info
3. [ ] **Review active sessions** — sign out all other sessions:
   - Gmail: myaccount.google.com → Security → Your devices → Sign out of all
   - Microsoft: mysignins.microsoft.com → Sign out everywhere
4. [ ] **Check connected apps** — revoke any OAuth grants you don't recognize
5. [ ] **Enable MFA** if not already enabled (the attacker may have gotten in because MFA wasn't on)
6. [ ] **Change passwords on any accounts that used the same password** (this is why you don't reuse)
7. [ ] **Check banking and financial accounts** for unauthorized transactions
8. [ ] **Alert your team** — if this is a business account, others may be targeted too
9. [ ] **Document everything** with timestamps — you may need this for insurance or law enforcement

## 6. The 5-Minute Credential Security Check

Don't have time for the full guide? Do these 5 things right now:

1. **Check haveibeenpwned.com** for your primary email — 30 seconds
2. **Enable MFA on your email** if not already on — 2 minutes
3. **Check if your banking has MFA enabled** — 1 minute
4. **Install Bitwarden** (free) on your phone and computer — 2 minutes
5. **Change any password you've reused** on a financial account — as long as it takes

That's it. Those 5 actions block the majority of credential-based attacks.
