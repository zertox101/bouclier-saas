# Contributing to Mythos Response

Thank you for wanting to help. The more people who contribute, the better prepared we all are.

## How to Contribute

### Report Corrections
If you find factual errors, outdated information, or broken links:
1. Open an issue with the label `correction`
2. Cite your source
3. We'll verify and update

### Add Information
If you have new intelligence, vendor guidance, or government advisories:
1. Fork this repo
2. Add or update the relevant document
3. Include source URLs and dates
4. Submit a pull request

### Translations
This information needs to reach non-English speakers. If you can translate:
1. Create a directory for your language (e.g., `docs/es/`, `docs/fr/`)
2. Translate any document
3. Submit a pull request

### Industry-Specific Guidance
We need response plans tailored to specific sectors:
- Healthcare (HIPAA, medical devices, unpatchable equipment)
- Legal (privilege protection, ethics obligations)
- Education (limited budgets, large networks)
- Local government (public services, legacy systems)
- Retail (POS systems, PCI compliance)
- Manufacturing (OT/ICS, IoT devices)

If you have expertise, create a new document in `docs/industry/` and submit a PR.

### Tool Recommendations
If you know of free or affordable security tools that help small businesses:
1. Add them to the relevant document
2. Be honest about limitations
3. Disclose any affiliations

## What We're Looking For

- **Accuracy over speed** — cite your sources
- **Actionable over theoretical** — checklists beat essays
- **Accessible over technical** — write for a non-technical business owner first, then add technical depth
- **Honest assessments** — don't hype tools, don't minimize risks

## What We DON'T Want

- Marketing or product placement disguised as guidance
- Classified or non-public vulnerability details
- Exploit code or weaponization guidance
- Content that helps attackers more than defenders
- AI-generated content that hasn't been human-verified

## Code of Conduct

Be helpful. Be honest. Be kind. We're all in this together.

## Questions?

Open an issue with the label `question`.
